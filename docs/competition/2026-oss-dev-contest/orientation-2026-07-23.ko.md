## 1차 평가 기준: 30점

각 항목은 6점이다.

1. **프로젝트 구조 및 코드 완성도**
   - 코드 가독성, 주석 활용, 구조의 합리성
   - 목적에 맞게 기능하는 정도
2. **오픈소스 프로젝트로의 발전 가능성**
   - 현재 오픈소스로 진행 중이거나 향후 발전할 수 있는 가능성
3. **개발 문서의 구체성**
   - 목적에 맞게 작성되었으며 쉽게 이해할 수 있는 문서
4. **프로젝트 혁신성**
   - 기술력과 기술 수준, 최신 기술 활용 정도
5. **프로젝트 팀워크**
   - 팀은 역할 분배와 운영에서 드러나는 팀워크 수준
   - 개인은 프로젝트 관리체계의 안정성과 체계성
   - GitHub Issues, review, pull requests, commit, merge, 커뮤니티 등의 관리체계

## 오리엔테이션에서 반복된 판단 기준

### 1. 코드 공개보다 협업 가능한 오픈소스

라이선스에 따라 사용·수정·재배포할 수 있고, 외부 기여자가 맥락을 이해해 참여할 수 있어야 한다. 저장소에는 최소한 프로젝트 목적, 설치·실행 방법, 라이선스, 기여 방법이 명확해야 한다. 사용한 외부 코드와 데이터의 출처 및 라이선스도 추적 가능하게 남긴다.

### 2. AI 시대에는 판단·검증·운영 기록이 차별점

AI가 만든 코드의 양보다 팀이 무엇을 문제로 정의했고, 어떤 제안을 채택하거나 거절했으며, 그 판단을 어떤 테스트와 지표로 검증했는지가 중요하다. 다음 맥락이 Issues와 Pull Requests에 남아 있어야 한다.

- 해결하려는 사용자 문제와 성공 기준
- 시도했지만 실패한 접근과 폐기한 이유
- AI 제안을 채택·수정·거절한 판단 근거
- 리뷰 의견과 반영 결과
- 실행 환경, 재현 절차, 알려진 한계

### 3. 검증 가능한 수치가 필요

정확도, 성공률, 비용, 지연시간 등 프로젝트에 맞는 지표를 정의한다. 테스트, 실행 로그, 재현 절차를 통해 주장과 결과를 연결한다. 이후 단계에서도 라이선스와 실제 기능 검증이 이어질 수 있으므로 1차 제출 전에 증거를 정리한다.

### 4. 저장소 자체가 운영 기록

Issues에서 문제와 완료 조건을 정의하고, Pull Requests에서 구현 선택과 리뷰를 남기며, 커밋과 병합으로 변경 이력을 연결한다. 결과보고서와 시연 영상은 이 저장소의 증거를 요약하고 보여주는 역할을 맡는다.

## HealthMes 적용 체크리스트

### 코드 및 재현

- [ ] 깨끗한 환경에서 설치부터 핵심 시나리오 실행까지 재현한다.
- [ ] 핵심 기능의 성공 기준과 테스트 결과를 기록한다.
- [ ] 데모 경로에 필요한 설정, 샘플 데이터, 실패 시 복구 방법을 문서화한다.
- [ ] 결과보고서, 소스코드, 시연 영상이 동일한 커밋을 기준으로 한다.

### 오픈소스 준비

- [ ] README에서 문제, 대상 사용자, 주요 기능, 빠른 시작을 설명한다.
- [ ] LICENSE와 외부 의존성·코드·데이터의 출처 및 라이선스를 점검한다.
- [ ] CONTRIBUTING에서 제안, 개발, 테스트, 리뷰 절차를 안내한다.
- [ ] 후속 로드맵과 외부 기여자가 시작할 수 있는 Issue를 마련한다.

### 판단과 운영 증거

- [ ] 주요 설계·제품 판단을 Issue 또는 Pull Request에 이유와 함께 남긴다.
- [ ] 실패한 접근과 대안을 삭제하지 않고 결정 맥락으로 보존한다.
- [ ] Pull Request에 테스트 결과, 리뷰, 수정 내역을 연결한다.
- [ ] AI 도움을 받은 변경도 사람이 확인한 범위와 검증 결과를 기록한다.

### 제출물 정합성

- [ ] 결과보고서의 핵심 주장마다 저장소 증거 링크를 연결한다.
- [ ] 시연 영상은 설치 설명이 아니라 핵심 사용자 문제 해결 흐름에 집중한다.
- [ ] 영상에서 보여준 결과를 동일한 커밋과 절차로 재현할 수 있게 한다.
- [ ] 제출 전 공식 공고와 포털에서 일정, 형식, 용량, 필수 항목을 다시 확인한다.

## Builder Case에서의 위치

오리엔테이션 참여와 이 문서는 **Builder 협업·준수 증거**다. A3 데모 작업을 변경하거나 대체하지 않으며, 출품 가능한 기능이나 사용자 traction으로 계산하지 않는다. 실질적인 진척은 별도의 구현 커밋, 테스트, 재현 가능한 데모, 리뷰 기록으로 증명한다.

녹음과 자동 전사 원문은 개인정보와 저장소 크기를 고려해 저장소 밖의 로컬 작업 공간에 보관한다.
